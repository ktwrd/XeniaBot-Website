First update for 2026, and we've already passed Xenia's 3rd Birthday! <sup>(time really does go faster the older you get)</sup>

Lots of things have changed in the backend with this release, and we've started to make the move to a new database engine and ORM. Because of this, development work should start to pickup again! 🥳

**Changes**
- Started the move to Entity Framework Core w/ PostgreSQL
  - BanSync is now using EF Core, so it should be much more snappy in the Web Panel.
- Started caching user/guild information (along with taking snapshots) to reduce the amount of calls to the Discord API
  - The Web Panel should feel significantly more responsive now. Before, Xenia was calling the Discord API every time it wanted the avatar for a user, but it now makes a database query instead. For large servers, the Mutual BanSync Records page should load in <1s. (assuming you're in Australia)
- Include more MIME types for compression in the Web Panel.
- User/Guild IDs are now visible when viewing mutual BanSync Records in the Web Panel
- Refactored error reporting code.
  - Fixed Discord models not serializing to JSON properly in DiscordCacheService when reporting errors.
- Upgraded to .NET 10
- Fixed Guild State checking when requesting BanSync (now checks for Log Channel permissions)
